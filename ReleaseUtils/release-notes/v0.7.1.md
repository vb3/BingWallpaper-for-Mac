Bug-fix release: notifications added in 0.7.0 couldn't actually be enabled. This fixes the underlying cause.

## What's fixed since v0.7.0
- **Notifications can now be enabled.** The 0.7.0 build was shipped effectively unsigned (linker-only ad-hoc, with the Info.plist unbound and the sandbox entitlements dropped). macOS won't let an app in that state register for notifications, so toggling "Notify when wallpaper changes" failed and BingWallpaper never appeared under System Settings → Notifications. Release builds are now properly ad-hoc signed with their entitlements (the same identity Xcode's "Sign to Run Locally" produces), so the permission prompt works and the app registers correctly.
- The release script now verifies the signature and entitlements after building and refuses to publish a build that isn't properly signed, so this can't regress.

If you installed 0.7.0, upgrade to 0.7.1, open Settings, and toggle "Notify when wallpaper changes" — macOS should now prompt you to allow notifications.

## Install
1. Download **BingWallpaper_v0.7.1.pkg** below and run it (installs to `/Applications` and launches the app).
2. **Gatekeeper note:** this build is signed ad-hoc (not notarized), so macOS may warn that the developer can't be verified. If so, **right-click the app → Open → Open**, or run:
   `xattr -dr com.apple.quarantine /Applications/BingWallpaper.app`

Or via Homebrew: `brew upgrade --cask bingwallpaper`.

Requires macOS 14 (Sonoma) or later.
