Maintenance release focused on reliability and efficiency. No new features or settings — existing behavior is unchanged, just more robust and lighter on resources.

## What's changed since v0.5.4
- **More reliable downloads** — the daily fetch now verifies the server response (HTTP status) and confirms the downloaded data is a valid image before saving, so a transient error page or bad response can no longer be stored as a wallpaper or corrupt the image list.
- **Lower background overhead** — the wallpaper is no longer re-applied to a display that already shows it (for example on every Space switch), cutting redundant work while still covering newly created Spaces and newly connected monitors.
- **Smoother menu preview** — the menu's image preview is reused instead of rebuilt every time the menu opens, and a slower image load can no longer overwrite a newer day you've just selected.

## Install
1. Download **BingWallpaper_v0.6.0.pkg** below and run it (installs to `/Applications` and launches the app).
2. **Gatekeeper note:** this build is signed ad-hoc (not notarized), so macOS may warn that the developer can't be verified. If so, **right-click the app → Open → Open**, or run:
   `xattr -dr com.apple.quarantine /Applications/BingWallpaper.app`

Or via Homebrew: `brew upgrade --cask bingwallpaper`.

Requires macOS 14 (Sonoma) or later.
