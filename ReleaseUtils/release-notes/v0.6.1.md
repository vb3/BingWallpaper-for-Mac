Maintenance release focused on robustness and a fix for duplicate launches. No new settings — existing behavior is unchanged, just more reliable.

## What's changed since v0.6.0
- **Single instance enforced** — launching a second copy of BingWallpaper no longer adds a duplicate menu-bar icon. The newcomer detects the running app, brings its Settings window forward, and exits.
- **Settings opens on re-launch, not on every activation** — re-opening the app surfaces Settings (handled via reopen), instead of popping it open on incidental activations.
- **Stronger image validation** — downloaded images are now fully decoded before saving, so a truncated or partial download can't be stored as a broken wallpaper; the missing image is simply re-fetched on the next update.
- **Corrupt files self-heal** — image writes are atomic, so a re-download cleanly replaces any leftover partial file.
- **Hardened cleanup** — the keep-last-N image cleanup only ever touches `.jpg` files and matches dates exactly.
- **Tidier captions** — image description/credit parsing handles titles that themselves contain parentheses.

## Install
1. Download **BingWallpaper_v0.6.1.pkg** below and run it (installs to `/Applications` and launches the app).
2. **Gatekeeper note:** this build is signed ad-hoc (not notarized), so macOS may warn that the developer can't be verified. If so, **right-click the app → Open → Open**, or run:
   `xattr -dr com.apple.quarantine /Applications/BingWallpaper.app`

Or via Homebrew: `brew upgrade --cask bingwallpaper`.

Requires macOS 14 (Sonoma) or later.
